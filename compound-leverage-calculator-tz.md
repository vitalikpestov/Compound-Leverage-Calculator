# Compound Leverage Calculator — техническое задание

Рабочее название продукта: **Compound Leverage Calculator**

Альтернативы для брендинга:
- **Capital vs Debt Calculator**
- **Net Capital Growth Calculator**
- **Equity & Debt Horizon Calculator**
- **Compound Equity Planner**

Рекомендуемое рабочее название: **Compound Leverage Calculator**. Оно отражает суть продукта: рост капитала по сложному проценту и параллельное накопление долговой нагрузки в одной модели.[cite:17][cite:24][cite:26]

## Описание продукта

Нужно создать одностраничный веб-калькулятор, который моделирует рост инвестиционного капитала, накопление долга от регулярных ежемесячных заимствований и рассчитывает итоговый чистый капитал на выбранном горизонте.[cite:17][cite:23][cite:24]

Продукт должен быть визуально похож на премиальный минималистичный fintech/web3-калькулятор: светлый фон, большие скругленные карточки, мягкие бордеры, чистая типографика, много воздуха, спокойные акцентные цвета.[cite:34][cite:35]

## Основная задача

Пользователь задаёт стартовый депозит, доходность капитала, горизонт расчёта, ежемесячный займ и ставку по займу. После каждого изменения калькулятор должен пересчитывать итоговые показатели, график и таблицу годовых срезов.[cite:17][cite:20][cite:24]

Продукт должен показывать не только итоговое значение, но и динамику во времени, чтобы пользователь видел, как сложный процент ускоряет рост капитала, а заемная нагрузка влияет на чистый капитал.[cite:39][cite:40][cite:48]

## Основные входные параметры

Поля ввода:
- Стартовый депозит (USD)
- Доходность Equity (%)
- Горизонт (лет)
- Ежемесячный займ (USD)
- Ставка по займу (%)

Требования к полям:
- только числовые значения;
- без отрицательных значений;
- поддержка вставки значений с пробелами и запятыми;
- при blur форматирование с разделителями тысяч;
- при focus показ «чистого» числа для удобного редактирования;
- мягкая валидация и защита от поломки интерфейса.[cite:17][cite:20]

## Выходные метрики

Нужно выводить 4 итоговые карточки:
- Капитал в конце
- Долг в конце
- Чистый капитал
- Всего займов

Все итоговые значения должны округляться до целых долларов только на уровне отображения, а внутри расчётов должна сохраняться полная точность JavaScript Number.[cite:24][cite:25]

## Расчётная логика

Базовая модель должна быть помесячной.

Обозначения:
- `initialDeposit` — стартовый депозит
- `equityRateAnnual` — годовая доходность капитала, %
- `loanRateAnnual` — годовая ставка долга, %
- `monthlyLoan` — фиксированная сумма займа в месяц
- `years` — горизонт расчёта
- `months` — количество месяцев, `years * 12`

Преобразования:
- `equityRateMonthly = equityRateAnnual / 100 / 12`
- `loanRateMonthly = loanRateAnnual / 100 / 12`

Инициализация:
- `capital = initialDeposit`
- `debt = 0`
- `totalBorrowed = 0`

Для каждого месяца от 1 до `months`:
1. `capital = capital * (1 + equityRateMonthly)`
2. `debt = debt * (1 + loanRateMonthly)`
3. `debt = debt + monthlyLoan`
4. `totalBorrowed = totalBorrowed + monthlyLoan`

Итог:
- `endingCapital = capital`
- `endingDebt = debt`
- `netCapital = endingCapital - endingDebt`
- `totalBorrowed = totalBorrowed`

Важно:
- ежемесячный займ не добавляется автоматически в инвестиционный капитал;
- капитал и долг считаются как две отдельные накопительные линии;
- расчёт строится по принципу monthly compounding с регулярными monthly additions для долга.[cite:17][cite:21][cite:23][cite:26]

## Контрольный пример

Для следующих значений:
- стартовый депозит = 100000
- доходность equity = 10
- горизонт = 20
- ежемесячный займ = 1000
- ставка по займу = 4

Ожидаемые результаты:
- Капитал в конце ≈ 672 750 $
- Долг в конце ≈ 365 033 $
- Чистый капитал ≈ 307 717 $
- Всего займов = 240 000 $

Допускается небольшое расхождение из-за округления, но логика расчёта должна совпадать.[cite:17][cite:24][cite:26]

## Структура интерфейса

### Верхняя панель

Справа расположить:

- быстрые пресеты горизонта: 5 лет, 10 лет, 20 лет, 30 лет.

Активный пресет должен быть визуально выделен как selected chip / segmented control.[cite:34][cite:41]

### Блок параметров

Содержимое:
- overline: «Параметры»
- title: «Настройте капитал, займ и горизонт расчёта»

Сетка полей:
- Стартовый депозит
- Доходность Equity
- Горизонт
- Ежемесячный займ
- Ставка по займу

Под каждым полем должен быть helper-text с коротким объяснением назначения параметра.

### Блок итогов

4 карточки результатов:
- Капитал в конце
- Долг в конце
- Чистый капитал
- Всего займов

На desktop — 4 карточки в ряд, на tablet — 2x2, на mobile — одна колонка.

## Новый блок: график роста капитала

### Назначение

Визуализировать во времени 3 линии:
- капитал;
- долг;
- чистый капитал.

График должен строиться на основе помесячной модели, а не только годовых точек, чтобы сохранялась точная форма compounding-кривой.[cite:40][cite:46]

### Заголовок блока

RU:
- Overline: «График»
- Title: «Капитал, долг и чистый капитал во времени»
- Helper: «Метки Rule of 72 отмечают уровни 2x, 4x, 8x от стартового депозита.»

EN:
- Overline: `Chart`
- Title: `Capital, debt, and net capital over time`
- Helper: `Rule of 72 markers highlight 2x, 4x, and 8x levels from the initial deposit.`

### Данные для графика

На каждом месяце нужно сохранять:
- `monthIndex`
- `yearFloat = monthIndex / 12`
- `capital`
- `debt`
- `netCapital = capital - debt`
- `totalBorrowed`

### Серии графика

Отрисовать 3 основные линии:
1. Capital
2. Debt
3. Net Capital

Цвета:
- Capital — зеленовато-teal
- Debt — warm amber / orange
- Net Capital — глубокий navy / charcoal

Над графиком расположить chips-переключатели видимости серий:
- Капитал
- Долг
- Чистый капитал
- Rule of 72

Legend chips должны быть keyboard-accessible и работать как toggles.[cite:34][cite:35][cite:41]

### Rule of 72

Если toggle включен, на линии капитала показать milestone-маркеры достижения уровней:
- 2x
- 4x
- 8x
- 16x
- 32x

Логика:
- базовый уровень = `initialDeposit`
- цели = `initialDeposit * [2, 4, 8, 16, 32]`
- для каждой цели найти первый месяц, где `capital >= targetLevel`
- поставить marker на фактической точке кривой

Это должно быть не отдельным прогнозом, а визуальными метками поверх реальной модели.[cite:39][cite:42][cite:48]

### Tooltip графика

При hover / focus показывать кастомный tooltip:
- период, например: `2 г 3 мес` / `2y 3mo`
- Капитал
- Долг
- Чистый капитал

Tooltip должен быть выполнен как отдельная HTML-card в стиле интерфейса, а не дефолтный canvas-tooltip.[cite:35][cite:36][cite:41]

### Оси графика

Ось X:
- время в годах;
- подписи RU: `1 г`, `2 г`, `3 г`;
- подписи EN: `1y`, `2y`, `3y`.

Ось Y:
- значения в USD;
- компактный формат отображения;
- RU: `0 $`, `500 тыс $`, `1,1 млн $`;
- EN: `$0`, `$500k`, `$1.1M`.

### Реализация графика

Можно использовать Chart.js. Для реализации важны:
- кастомная HTML legend;
- кастомный tooltip;
- аккуратное поведение interactions;
- чистая отрисовка line chart без перегруженности.[cite:34][cite:35][cite:37][cite:41]

## Новый блок: таблица годовых срезов

### Назначение

Показать годовые snapshot-срезы из помесячной модели, чтобы пользователь видел контрольные точки на конец каждого полного года.[cite:43][cite:46]

### Заголовок блока

RU:
- Overline: «Таблица по годам»

EN:
- Overline: `Annual table`

### Логика таблицы

Для каждого года N брать состояние на конец каждого полного года:
- 1 год = после 12 месяцев
- 2 год = после 24 месяцев

Для каждого ряда показывать:
- `year`
- `monthlyLoan`
- `capital`
- `debt`
- `netCapital`
- `totalBorrowed`

Если горизонт = 20 лет, таблица должна содержать 20 строк.

### Колонки таблицы

- Год

- Капитал
- Долг
- Чистый капитал
- Всего займов

EN:

### UX таблицы

## Валидация

Ограничения:
- стартовый депозит >= 0
- доходность equity: 0..100
- горизонт: 1..50
- ежемесячный займ >= 0
- ставка по займу: 0..100

При выходе за диапазон:
- либо clamp;
- либо inline validation;
- интерфейс не должен ломаться.

## Accessibility

Требования:
- semantic HTML;
- `label` для каждого input;
- `aria-live="polite"` для итоговых метрик;
- заметные focus state;
- доступность legend chips с клавиатуры;
- графику нужен text summary для screen readers;
- таблица должна быть корректной semantic table.[cite:34][cite:35][cite:37]
